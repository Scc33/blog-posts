Six Vocab Words to Get Started With Kubernetes

Kubernetes is confusing but it doesn’t have to be if you understand a few key words

Kubernetes is an open-source platform designed to make managing codebases easier and more deployable. It is a way of standardizing code to be able to scale easily and run anywhere. Kubernetes is one of the most popular programming technologies out there and it comes with a lot of new vocabulary. However, with just a few key definitions it is easy to get started.

Container

The whole idea of Kubernetes is based around containerization. A containers is an executable that contains the running software and its dependencies. It is designed to abstract away the physical infrastructure that will run the code.

Cluster

A cluster is a set of nodes that run your application. Kubernetes says every cluster will have at least one node worker.

Node

Simply put a node is the worker machine in Kubernetes. It can be either a virtual or physical machine. It is managed by the API and contains the needed dependencies to run pods.

Pod

Is the smallest deployable object in the Kubernetes ecosystem. “A pod represents a set of running containers on your cluster.”

Kubernetes API

An application built on a RESTful interface that provides Kubernetes functionality and maintains the state of the cluster. It can be thought of as the lifeblood of Kubernetes.

Kubectl

Is the command line tool (or command line interface, CLI) that manages communication with the Kubernetes cluster’s. Kubectl uses the Kubernetes API.
