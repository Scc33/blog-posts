## The Problem

[https://leetcode.com/problems/unique-binary-search-trees/](https://leetcode.com/problems/unique-binary-search-trees/)

> Given an integer `n`, return *the number of structurally unique* ***BST'****s (binary search trees) which has exactly* `n` *nodes of unique values from* `1` *to* `n`.

This is a LeetCode Medium style problem.

## The Solution

### **What is Dynamic Programming?**

Before diving into the problem solution, it's important to briefly touch upon what Dynamic Programming (DP) is. DP is a technique used to solve problems by breaking them down into smaller sub-problems and storing the solutions to those sub-problems to avoid redundant work. The core idea behind DP is quite simple: save the answers to sub-problems you've already solved so that you can re-use them later.

### Solution in Code

```python
def numTrees(self, n):
        dp = [0] * (n+1)
        dp[0] = 1
        for i in range(1, n+1):
            for j in range(i):
                dp[i] += dp[j] * dp[i- 1 - j]
        return dp[n]
```

The most effective way to approach this problem is by using Dynamic Programming. The basic idea is to utilize smaller problems (i.e., a smaller number of nodes) to solve larger ones. For a given node `i`, you can put it as the root, and all the smaller numbers will go on the left subtree, and all the larger numbers will go on the right subtree.

Let `dp[i]` be the number of unique BSTs that you can create with `i` nodes. To calculate `dp[i]`, you iterate through each `j` from 1 to `i`, considering `j` as the root. Then, you can multiply the number of unique BSTs with `j-1` nodes for the left subtree (`dp[j-1]`) and `i-j` nodes for the right subtree (`dp[i-j]`). Summing these up for all `j` gives you `dp[i]`.

### Time Complexity

The time complexity of this solution is `O(n^2)`, and it requires `O(n)` extra space for storing the DP table.
